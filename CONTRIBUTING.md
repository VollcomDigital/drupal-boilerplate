# Contributing

Thanks for contributing.

## Development

1. Fork the repository.
2. Create a feature branch.
3. Run local checks and tests.
4. Open a pull request with a clear change summary.

## Commit Style

Use semantic commit prefixes when possible:

- `feat:`
- `fix:`
- `perf:`
- `chore:`
- `docs:`

## Security

If your report is security-sensitive, follow [SECURITY.md](SECURITY.md) and avoid public disclosure.
